# Shinylive PDF Signer

A mobile-first Python Shiny/Shinylive app for adding a visible handwritten signature, typed name, date, and optional text to a PDF.

## What it does

- Runs in the browser; the selected PDF is not uploaded to an application server.
- Lets the user draw and save a signature.
- Renders multi-page PDFs and allows signature/name/date/text placement on any page.
- Placed items can be dragged, resized, deleted, and undone.
- Creates a new signed PDF in the browser.
- Uses the iPhone/iPad share sheet to send the signed file to Mail or save it to Files.

## Important iPhone limitation

Safari can share the finished PDF to Mail, but it cannot inject an attachment into an already-open Mail reply thread. The reliable same-thread workflow is:

1. From Mail, save the incoming PDF attachment to Files.
2. Open this app and choose the saved PDF.
3. Sign it and tap **Prepare signed PDF**.
4. Tap **Share or Save** and choose **Save to Files**.
5. Return to the original Mail message, tap Reply, and attach the signed PDF from Files.

Choosing Mail in the share sheet starts a new draft rather than attaching to the existing thread.

## Run locally

```bash
python -m pip install shiny
shiny run --reload app.py
```

## Export as a Shinylive static site

```bash
python -m pip install shinylive
shinylive export . ../site
python -m http.server --directory ../site 8008
```

Open `http://localhost:8008` and deploy the generated `site` folder to GitHub Pages, Netlify, or another static host.

## Architecture

The interface is generated by Python Shiny. PDF rendering, touch placement, signature drawing, PDF modification, and iOS sharing run client-side in JavaScript because these browser APIs are the practical way to support iPhone gestures and file sharing in Shinylive.

The app loads pinned versions of:

- PDF.js 4.10.38 for page rendering
- pdf-lib 1.17.1 for adding signature marks and exporting the result

The app requires an internet connection to load these pinned libraries. They can be downloaded into `www/` and imported locally for a fully self-contained deployment.

## Limitations

- This creates a visible electronic signature mark, not a certificate-backed cryptographic digital signature.
- Password-protected/encrypted PDFs are not supported.
- Very unusual crop boxes or malformed PDFs may render differently from standard PDFs.
- Users should inspect the generated PDF before sending it.
